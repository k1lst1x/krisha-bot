# IMPLEMENTATION - Детали реализации

## Безопасная стартовая архитектура

Сейчас проект запускается в режиме `dry-run` по умолчанию. Это намеренно: до ручной проверки формы сообщений нельзя писать код массовой отправки, потому что архитектура зависит от того, как krisha.kz защищает форму.

Реальная отправка включается только после фазы 0 и реализации `KrishaMessenger.send_message(..., dry_run=False)`.

---

## screenshotter.py

**Цель:** визуальная диагностика страниц, которые открывает бот.

**Как работает:**
- Запускает Chromium через Playwright
- Открывает те же страницы категорий, что и парсер
- Может открыть страницу входа `https://krisha.kz/my`, которая редиректит на `id.kolesa.kz/login`
- Сохраняет PNG в `screenshots/YYYYMMDD-HHMMSS/`
- По умолчанию браузер headless, но можно включить видимый режим через `--show-browser`

**Команда:**

```bash
python main.py --dry-run --screenshots --limit 1
python main.py --dry-run --auth-screenshot --limit 0
```

## scraper.py

**Цель:** получение объявлений с krisha.kz.

**Как работает:**
- Формирует URL вида `https://krisha.kz/arenda/kvartiry/{city}/`
- Добавляет query-параметр `das[who]=1` для фильтра "частник"
- Добавляет `page=N` при пагинации
- Парсит HTML через BeautifulSoup/lxml
- Извлекает поля:
  - `listing_id`
  - `title`
  - `price`
  - `district`
  - `url`
  - `owner_name`, если есть на странице списка
  - `description`, если включен детальный парсинг
- Останавливает пагинацию, когда страница вернула меньше `page_stop_below` объявлений

**Статус:** Базово реализовано.

**Риск:** селекторы нужно сверить с текущей живой версткой krisha.kz.

## messenger.py

**Цель:** авторизация и отправка сообщений.

**Текущий статус:** реальная отправка не реализована. В `dry-run` возвращается успешная заглушка без сетевого POST.

**Текущий вход krisha.kz:**
- `/my` редиректит на `https://id.kolesa.kz/login/?destination=...`
- форма содержит `input name="login"` с placeholder `Введите номер телефона`
- email в поле логина на текущей форме приводит к сообщению `Регистрация возможна только по номеру телефона`
- в `.env` используется `KRISHA_LOGIN`, ожидаемый формат - номер телефона

**Что нужно проверить в фазе 0:**
- Можно ли отправить сообщение обычным POST без JS
- Есть ли CSRF-токен
- Есть ли Captcha
- Нужен ли WebSocket/GraphQL/ajax endpoint
- Разрешено ли это правилами krisha.kz для аккаунта

**Риск:** массовая автоматическая отправка может привести к блокировке аккаунта.

## ai_generator.py

**Цель:** генерация персонализированных сообщений.

**Как работает:**
- Принимает `Listing`
- Формирует короткий prompt с типом, ценой, районом и целью контакта
- Если есть `OPENAI_API_KEY`, вызывает OpenAI Responses API
- Если ключа нет или API недоступен, возвращает локальный fallback-шаблон
- Ограничивает длину ответа через `max_message_chars`

**Модель:** задается в `config.json` через `openai_model`.

## contacted.db

SQLite создается автоматически при первом запуске.

**Таблица `contacts`:**

| поле | тип | описание |
|------|-----|----------|
| listing_id | TEXT PK | id объявления |
| sent_at | DATETIME | когда написали |
| message | TEXT | что написали |
| response | TEXT | ответ, если есть |
| status | TEXT | `sent`, `failed`, `dry_run`, `replied` |
| listing_url | TEXT | ссылка на объявление |

## main.py

**Команды:**

```bash
python main.py --dry-run
python main.py --send
python main.py --limit 5
```

По умолчанию запуск работает как dry-run. Для реальной отправки нужен явный флаг `--send`, но сейчас он завершится ошибкой до реализации фазы 0.
